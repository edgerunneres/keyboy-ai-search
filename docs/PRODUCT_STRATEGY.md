# KeyBoy 产品差异化策略

## 1. 核心定位

KeyBoy 不再定位为“课程设计搜索引擎”，而是定位为“前沿项目雷达 + 可信研究决策工作台”。

用户不是缺一个搜索框，而是缺一个能把 GitHub、论文、开放学术源和本地资料转成可执行判断的工具。KeyBoy 的价值应当落在三个问题上：

- 我应该相信哪些资料？
- 我应该选择哪条技术路线？
- 下一步应该具体做什么？

## 2. 用户为什么会选择 KeyBoy

| 用户需求 | 普通搜索/RAG 的问题 | KeyBoy 的回答 |
| --- | --- | --- |
| 快速理解前沿方向 | 链接太多，需要自己判断 | 自动规划、检索、排序、合成、批判校验 |
| 做课程设计/项目答辩 | 只有结果，没有过程证据 | 展示 Agent Trace、引用、指标、风险 |
| 做技术选型 | 容易被 star 数或宣传影响 | 给出推荐路线、取舍判断、可信度评分 |
| 本地稳定演示 | 在线模型和网络不稳定 | 在线研究 + 本地 fallback 双模式 |
| 继续迭代产品 | 总结完就结束 | 给出下一步行动和可追问问题 |

## 3. 已融合的前沿项目优点

| 前沿项目/方向 | 优点 | KeyBoy 融合方式 |
| --- | --- | --- |
| DeerFlow / SuperAgent | 长任务拆解、子智能体、工具、记忆、沙箱 | 研究计划、Agent Trace、下一步行动队列 |
| Tongyi DeepResearch | 面向复杂研究任务的 agentic information seeking | 在线开放源检索、证据合成、批判校验 |
| GraphRAG / LightRAG | 图结构增强、概念关系、全局/局部检索 | 轻量知识地图、概念-来源关系、后续可升级图谱 |
| STORM / Co-STORM | 多视角调研、引用报告、人机协同知识整理 | 研究答案、继续追问、引用证据 |
| RAGFlow / Onyx | 工程化 RAG、文档理解、连接器、可追溯 | 在线源 + 本地知识库、质量指标、稳定兜底 |
| Self-RAG / Critic | 自我批判、按需检索、不确定性控制 | CriticAgent、风险提示、可信度评分 |

## 4. 当前版本的特色功能

- 决策简报：把检索结果转成用户需求、推荐路线、选择 KeyBoy 的理由和取舍判断。
- 可信度评分：用证据数量、来源多样性、资料新鲜度、模型状态和风险扣分计算可信等级。
- 知识地图：从 Top-K 证据抽取概念和来源覆盖，让用户看到问题的知识结构。
- 继续追问：把研究结论转成下一轮可执行问题，形成研究闭环。
- 前沿能力融合面板：让用户知道 KeyBoy 吸收了哪些强项目的思路，以及如何落到当前系统。

## 5. 下一步优先级

1. 接入网页正文阅读：对 GitHub README、论文页和技术博客做正文抽取。
2. 增强知识地图：从关键词升级为实体、关系和社区摘要。
3. 增加项目雷达：定期扫描 GitHub 项目热度、更新时间、issue 健康度和技术标签。
4. 增加报告导出：把答案、引用、图谱、风险和下一步行动导出为课程设计报告。
5. 增加用户工作区：保存研究任务、对比不同路线、持续跟踪新资料。
